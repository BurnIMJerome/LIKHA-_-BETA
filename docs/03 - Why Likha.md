# Why Likha

Likha exists for teams that want practical automation without being forced into a large platform commitment on day one.

## Problems Likha Targets

- RPA platforms can become expensive before the first bot is in production.
- Small teams often need a focused desktop/browser automation tool before they need a full enterprise program.
- AI automation should work with the provider the team already uses.
- Some organizations need on-premise or private infrastructure instead of mandatory cloud orchestration.
- Developers need clear activity behavior, readable workflow data, and direct control over runtime setup.

## Likha's Approach

- Build locally first.
- Run interactively in the designer when developing.
- Queue only when running unattended or through schedules/triggers.
- Use Control Room when coordination, queues, logs, schedules, and robot visibility matter.
- Let users bring their own AI settings.

## What Likha Is Not Trying To Be

Likha is not trying to copy every feature of mature enterprise RPA suites. It focuses on common automation work: desktop, browser, Excel, files, APIs, queues, scripts, data, AI, and orchestration basics.
